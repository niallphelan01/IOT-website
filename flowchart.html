Flowchart of the system running

Main parts of the flow

  • The alarm system is started via a virtual button from Blynk
  • The alarm takes 5 seconds to set, leaving the user leave the shed
  • Once the alarm is set the status is updated to Wia and Blynk via MQTT
  • Once the alarm is set I use the aysynchronous nature of node js to watch for a change in the status of the sensors (door, tamper and PIR)
  • If the alarm is triggered then the pi camera records image and video and this will be emailed via a Wia event
